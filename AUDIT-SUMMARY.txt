Date: 2026-06-02

Scope
- Audited current GitHub Pages app and source for role workflows, visible buttons, cloud sync, mobile scrolling, PWA behavior, and Apps Script data integrity.
- Tested the deployed Apps Script endpoint with a read-only readAll request.
- Tested local source syntax for all inline scripts, sw.js, and backend/Code.gs.

Broken functionality found and fixed
1. Student Edit action
- Problem: one patched student table rendered Edit buttons that called openEditStudent(), but only editStudent() existed.
- Root cause: late runtime patch introduced a renamed click target without an alias.
- Fix: added window.openEditStudent() as a safe alias to editStudent().

2. Cloud read data loss
- Problem: backend readSheet() filtered out rows without row.id. Attendance and instructor attendance rows can exist without id, so cloud reads could silently drop attendance history.
- Root cause: id-only row filter was applied to every sheet schema.
- Fix: backend now keeps any non-empty row and includes explicit schemas for all app datasets.

3. Google Sheets date/time normalization
- Problem: Google Sheets returned times/dates as Date values such as 1899-12-30T17:00:00.000Z, which the UI treated as display/session strings.
- Root cause: Apps Script read path returned raw Date values and the frontend had no final normalizer for cloud data.
- Fix: backend normalizes time fields to HH:mm and date fields to yyyy-MM-dd; frontend also normalizes cloud data after sync.

4. Marketplace workflow gaps
- Problem: products could be created but not edited/deleted from the visible UI; parent buy buttons could be hidden by role CSS.
- Root cause: marketplace was implemented as simple cards without admin management actions, and broad role CSS hid primary buttons for parents.
- Fix: added product edit/delete controls for admin/manager, order listing, stock decrement on order, and parent-visible buy actions.

5. Event workflow gaps
- Problem: events could be created but not edited/deleted from the visible UI; parent registration actions could be hidden by role CSS.
- Root cause: event cards lacked edit/delete actions and inherited broad parent button hiding.
- Fix: added event edit/delete controls for admin/manager and parent-visible register actions.

6. Wallet transaction safety
- Problem: wallet transactions allowed zero/blank amounts and debits could exceed the balance.
- Root cause: walletTxn() did not validate amount or available balance.
- Fix: added amount validation, parent wallet linking checks, and insufficient-funds prevention.

7. PWA stale app shell
- Problem: service worker used cache-first behavior for the app shell, which could keep serving old index.html after repairs.
- Root cause: index.html was cached like a static asset.
- Fix: bumped cache to han-tkd-v7-audit-repair and made navigation/index requests network-first with offline fallback. Apps Script requests remain network-only.

8. Mobile login clipping
- Problem: a later scroll override could make the auth shell render too wide or too close to the viewport edge on phone-sized captures.
- Root cause: global scroll repair CSS changed auth overlay layout characteristics.
- Fix: added final auth-specific viewport constraints and verified the login screen renders fully in a 390x844 Edge screenshot.
